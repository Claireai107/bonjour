## 2. 사용자 플로우 (현재)

```
스플래시 영상 (세션당 1회, 탭으로 스킵)
  → 스플래시 화면 [시작하기]
  → 회원가입 (휴대폰·이름·성별·생년월일 휠·주소)  ※ 데모용 빠르게 채우기 버튼
  → 홈(메인)
  → [AI 뼈건강 분석 시작]
  → 설문 1~10번 (터치/음성, 디폴트값으로 [다음] 기본 활성)
  → 건강검진 입력 (사진 OCR / 직접 입력 / 건너뛰기)
  → AI 분석 (진행 링)
  → 리포트 (점수·또래비교·위험요인·개선 가능 점수)
  → 시뮬레이터 → 맞춤 루틴
```

- 하단 탭: **홈 · AI 루틴 · 우리동네 · 리포트 · 마이** (설문·회원가입 화면 제외 전 페이지 고정)
- 가족 프로필: 마이페이지 우측 상단 사용자 선택 → 전환/추가

## 3. 주요 기능 상세

### 3-1. 검진표 사진 인식 (OCR)
- 카메라 화면에서 **실시간 미리보기**(HTTPS/localhost) 또는 네이티브 카메라(폰 HTTP)
- 3단 폴백: **① Claude 비전(유료·최고성능) → ② Gemini 비전(무료, 현재 활성) → ③ tesseract(오프라인)**
- EXIF 회전 보정 + 다운스케일 전처리 (폰 세로 사진 대응)
- **참고치(정상범위) 숫자 무시하고 '결과' 값만 추출**
- 못 읽은 항목은 "직접 입력" 배지 → 탭해서 수정
- 검증: 참고치 포함 검진표 4/4 인식, 세로 사진 4/4

### 3-2. 위치 기반 (카카오 로컬 API)
- 회원가입: **[현재 위치]** 버튼 → GPS → 실제 주소 자동 입력
- 회원가입: **[주소 검색]** → 카카오 우편번호 서비스 (키 불필요)
- 우리동네/루틴: **[내 위치로 가까운 보건소 찾기]** → 거리·도보시간 순 표시, "자세히 보기" → 카카오맵
- 키 없으면 순천 예시 데이터로 폴백 (발표 안전장치)
- ⚠️ 폰에서 GPS는 HTTPS 필요 (로컬 http에선 데스크톱만 동작 → Vercel 배포 시 해결)

### 3-3. 설문
- 숫자 문항 디폴트: 나이 65 · 키 160 · **몸무게 70** · 폐경연령 50 · 초경연령 14 → **[다음] 기본 활성화**
- +/− 버튼 **길게 누르면 빠르게 증감** (0.45초 후 초당 ~14)
- 선택형: 답 선택 후 [다음] 눌러야 이동 (자동 넘김 제거)
- 건너뛰기 제거 (데이터 완결성)
- 음성 모드: 본이가 읽어주고 말로 답변 (설문 전 답변방식 선택)

### 3-4. 리포트
- **우측 상단 날짜 드롭다운**: 분석할 때마다 이력 저장(프로필당 최근 20건), 과거 리포트 열람. 디폴트 = 최신
- 실제 예측 모델 값 바인딩 (점수·또래 백분위·위험/보호 요인·개선 가능 점수)

### 3-5. 스플래시 영상
- `design/스플래시화면.MOV` → 웹용 MP4(H.264) 변환, `public/splash.mp4`
- 앱 실행 시 전체화면 자동재생 → 종료/탭 시 스플래시 화면 전환
- 세션당 1회만 재생, 로드 실패 시 자동 스킵

## 4. 외부 API / 환경변수 (`.env.local`)

| 키 | 용도 | 상태 | 비고 |
|---|---|---|---|
| `GEMINI_API_KEY` | 검진표 OCR (Gemini 비전) | ✅ 설정됨 | 무료 티어 · aistudio.google.com |
| `KAKAO_REST_API_KEY` | 주소 변환·보건소 검색 | ✅ 설정됨 | 무료 쿼터 · 카카오맵 사용설정 ON 필요 |
| `ANTHROPIC_API_KEY` | 검진표 OCR (Claude, 최우선) | ⬜ 미설정 | 유료 · 실서비스 전환 시 권장 |

- 모든 키는 **서버에서만 사용** (클라이언트 노출 없음), 키가 없어도 앱은 폴백으로 동작
- ⚠️ Gemini 무료 티어는 입력 데이터가 학습에 쓰일 수 있음 → 실사용자 검진표 단계에선 유료 전환 권장

## 5. 실행 방법

```bash
cd bonjour
npm install
npm run dev          # 개발: http://localhost:3000
# 또는
npx next build && npx next start -p 3100   # 프로덕션 모드
```

- 같은 와이파이 폰 접속: `http://<맥IP>:3100` (예: 192.168.0.247)
- 데모 데이터: 회원가입/프로필 추가의 **"데모용 빠르게 채우기"** (김순자 56세 / 이영희 64세)
- 카메라 미리보기·GPS는 HTTPS 필요 → Vercel 배포 시 폰에서 전부 동작

## 6. 프로젝트 구조 (주요)

```
bonjour/
├─ app/
│  ├─ page.tsx            # 스플래시 (영상 인트로 포함)
│  ├─ signup/             # 회원가입 (위치·우편번호·데모채움)
│  ├─ home/               # 홈 (분석 시작 / 오늘의 루틴)
│  ├─ onboarding/         # 답변 방식 선택 (터치/음성)
│  ├─ survey/             # 설문 10문항 (터치+음성)
│  ├─ checkup/            # 검진 입력 (촬영→인식→확인 / 직접)
│  ├─ analysis/           # AI 분석 로딩 (진행 링)
│  ├─ report/             # 리포트 (+ 날짜 이력 드롭다운)
│  ├─ simulator/          # What-if 시뮬레이터
│  ├─ routine/            # 맞춤 루틴 + 우리동네
│  ├─ local/              # 우리동네 탭
│  ├─ favorites/          # 관심 건강 관리
│  ├─ mypage/             # 마이페이지 (우상단 사용자 전환)
│  ├─ profile-add/        # 가족 프로필 추가
│  └─ api/
│     ├─ ocr/             # 검진표 OCR (Claude→Gemini)
│     ├─ health-centers/  # 주변 보건소 (카카오 로컬)
│     └─ reverse-geocode/ # 좌표→주소 (카카오)
├─ components/            # Boni·TabBar·ProgressBar·ScreenFrame·LocalSection·PostcodeSearch 등
├─ lib/                   # store(zustand)·predict·survey·geo·useSpeech 등
└─ public/                # boni-*.png · logo-vertical.png · splash.mp4
```

- 상태: zustand + sessionStorage persist (가족 프로필별 답변/결과/이력/관심)
- UI 규칙: `docs/UI_체계.md` 참고

## 7. 알려진 제약

| 항목 | 내용 |
|---|---|
| 폰 GPS/카메라 미리보기 | HTTP 접속에선 브라우저가 차단 → HTTPS(배포) 필요 |
| 음성 인식 | 크롬/엣지 권장 (iOS 사파리는 제한적) |
| OCR 첫 실행 | tesseract 폴백 시 한국어 데이터(~10MB) 최초 1회 다운로드 |
| 데이터 저장 | 기기(sessionStorage)에만 저장 — 서버 DB 없음 (데모 스펙) |
| 제휴 시설 카드 | 순천 필라테스/튼튼 헬스장은 데모 고정 데이터 |
